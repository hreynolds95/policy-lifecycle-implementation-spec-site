DMS Glossary and Translation Layer

A plain-English bridge between GitHub implementation terms and compliance policy operating language. Use this when stakeholders ask what the system means in practice, not what the engineering feature is called.

{% include site-nav.html %}
Translation Principle

Translate GitHub mechanics into governance outcomes.

The implementation can use GitHub terms internally, but stakeholder conversations should stay anchored to familiar DMS concepts: source of truth, approval evidence, role gates, publication controls, and audit traceability.

Messaging Rule

Lead with the compliance meaning before the platform term.

Say this first "This is the approval record."

Then explain that GitHub stores it as a pull request with comments, approvals, and checks.

Avoid leading with "Open a PR against main."

That is implementation language. It is accurate, but it is not the stakeholder mental model.

Anchor back to "Does this preserve PoP control intent?"

The design should be judged by governance parity, auditability, and operational clarity.

Glossary

Core translations for DMS conversations

GitHub Term DMS Meaning Say It This Way Why It Matters
Repository Governed document system of record. The controlled library where source documents and evidence live. Defines the authoritative home that replaces LogicGate for in-scope policies and standards.
Markdown Machine-readable policy or standard source file. The editable source record behind the published PDF. Allows validation, search, automation, AI indexing, and consistent publication output.
Metadata / front matter Required document registry fields. The structured record of owner, tier, review date, domain, and status. Drives routing, reminders, reporting, portal filtering, and migration checks.
Branch Controlled draft workspace. A temporary editing copy used before the approved source record changes. Lets multiple changes be reviewed without overwriting the published record.
Pull request Formal change and approval package. The place where redlines, comments, checks, and approvals are reviewed. Creates a durable evidence trail for the proposed change.
CODEOWNERS Role-based required reviewer mapping. The rule that routes a document to the right approving group. Supports tiered approval parity and prevents bypassing required reviewers.
Branch protection Merge gate enforcement. The control that blocks publication until required reviews and checks pass. Turns PoP tollgate requirements into enforceable workflow rules.
GitHub Actions Automated governance checks. The system-run checks for metadata, formatting, review dates, and publication readiness. Reduces manual QC and creates evidence that the control ran.
Issue Assignable work item or governance follow-up. The tracked record for a blocker, review reminder, exception renewal, or decision. Turns open work into visible, assignable, auditable follow-up.
Label Status, category, or routing marker. The tag that tells the system and team what kind of work this is. Supports filtering, reporting, automation triggers, and portfolio visibility.
Merge commit Approved version snapshot. The permanent record of exactly what changed and when it became authoritative. Connects approved content to the publication and audit evidence chain.
GitHub Pages Staff-facing policy portal surface. The readable website experience generated from the governed source records. Keeps employee access simple while governance work remains controlled in GitHub.
Release / artifact Published document output. The generated PDF or portal file created from the approved source. Prevents disconnected PDFs by tying each output back to a commit.
Audit log Evidence register. The retained record of publication events, approvals, exceptions, and control outcomes. Supports regulator-ready traceability from output back to source and approval.
Shortcut Language

Recommended phrases for stakeholder conversations

Instead of "PR" Change package

Use when explaining approvals, redlines, QC checks, and stakeholder comments.

Instead of "main branch" Approved source record

Use when explaining why direct editing of published content is blocked.

Instead of "GitHub Action" Automated control check

Use when explaining format validation, metadata validation, and publication readiness.

Instead of "merge" Approve into the source of truth

Use when explaining the moment a change becomes authoritative.
