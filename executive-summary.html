Decision Ask

Approve Phase 2 readiness work for a GitHub-native DMS replacement.

Approve GitHub as the target system of record.

Proceed on the assumption that GitHub fully replaces LogicGate for governed policies and standards.

Confirm PoP approval parity requirements.

Resolve which tollgates, approval tiers, QC checks, and exception paths must be enforceable before cutover.

Assign DRIs for critical workstreams.

Assign owners for architecture, access control, governance workflow, migration, and open decisions.

Proposal Summary

What changes in the target operating model

Source of Truth Policies and standards live in GitHub.

Markdown source files, metadata, approvals, commits, PDFs, and audit logs remain traceable in one system.

Two Surfaces Employees use a portal; governance teams use GitHub.

The front door stays simple while controlled change, approvals, and evidence remain enforceable.

Approval Parity PoP tollgates are translated into GitHub controls.

Branch protection, CODEOWNERS, issue records, checks, and labels support approval routing and merge gates.

Automation Recurring governance checks become scheduled evidence.

Review reminders, exception expiry, orphan detection, and portfolio digests become structured GitHub records.

Risks To Resolve

Cutover depends on a few high-confidence decisions.

Approval model

Tiered approval parity must be mapped clearly enough to enforce in GitHub without ambiguity.

Legacy evidence

LogicGate history, publication records, and archive references must be migrated or formally accepted as gaps.

Portal scope

Staff-facing access must remain simple, even if governance teams operate in GitHub.

Operating ownership

Metadata quality, exception renewal, orphan detection, and monthly portfolio review need named owners.
