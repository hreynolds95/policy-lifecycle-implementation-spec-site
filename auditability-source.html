DMS Auditability and Source of Truth

The DMS makes GitHub the authoritative record for governed document content, approvals, publication evidence, and audit traceability. The published PDF is an output of the source record, not a disconnected file.

Source of Truth

Markdown in GitHub is the governed source record.

The repository stores policy and standard content in a version-controlled, machine-readable format. Approvals, comments, checks, and publication events attach to that record through pull requests, commits, workflow logs, and audit entries.

Evidence Chain

Every published document should trace back to one commit.

01 Markdown source

Policy content, metadata, approval history, and review dates live in the repository.

->
02 Pull request record

The PR contains the redline, comments, approvals, checks, and tier routing evidence.

->
03 Merge commit

The approved change lands as a permanent version-controlled snapshot.

->
04 Published PDF

The Block-branded PDF is generated from the merged source and tied to the commit.

->
05 Audit log

The publication event, version, evidence links, and notification record are retained.

Audit Record

What an auditor or regulator should be able to verify

Content

The exact policy or standard text that was approved and published.

Authority

The reviewers, approval label, tier path, and policy governance gate that allowed merge.

Time

The commit timestamp, publication date, review date, and notification timing.

Output

The generated PDF, portal record, and any Google Drive archive reference.

Controls

The validation checks, role enforcement, routing, and approval controls that ran.

Follow-up

Open exceptions, annual review reminders, owner reassignment, or corrective actions.

Why This Matters

The system avoids version confusion and disconnected evidence.

No file drift

The source document, approval trail, and published output remain tied together.

Machine-readable corpus

Markdown can be validated, searched, indexed by Quincy, and used by future AI tooling.

Regulator-ready traceability

A PDF can be shown with the commit and workflow history that produced it.
