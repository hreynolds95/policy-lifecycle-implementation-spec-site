Workflow

  1. Requester opens exception issue with document reference, requirement, justification, mitigating controls, effective date, expiry date, and evidence.
  2. Action reads `document-registry.yaml` and assigns the Document Owner.
  3. Owner approval moves the issue to `exception-approved` and records the decision.
  4. Workflow commits an exception record to the document package and notifies Policy Team.
  5. Daily cron flags upcoming expiry and opens renewal issues.

Exit Criteria

  • Required fields cannot be bypassed.
  • Evidence attachment is retained or linked.
  • Expiry alerting produces test issues in a staging repo.
  • Exception records are visible in audit exports.
