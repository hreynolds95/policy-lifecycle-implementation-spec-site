DMS Role-Based Experience

The DMS is designed so each group only touches the surface they need. Staff use the portal to find documents. Owners and submitters use one governed pull request. Policy Governance controls routing, approvals, exceptions, and the registry.

Role Model

The experience changes by responsibility, not by org chart.

Most employees consume published documents and never need GitHub. The people accountable for document changes get a structured governance path with enough control to preserve approval evidence, version history, and auditability.

Role 01

All Block Employees

Portal-only access for finding current compliance documents without GitHub training.

Primary surface
Policy Library Portal
GitHub required?
No
Training burden
Low: search, filter, read, request.
  • Search and filter documents by domain, tier, owner, and review status.
  • See overdue, coming due, and under-review documents at a glance.
  • Open published PDFs and ask Quincy natural-language questions.
Role 02

Document Owners + Submitters

Portal initiation plus one governed GitHub pull request for document changes.

Primary surface
Portal + GitHub PR
GitHub required?
Yes, for governed changes
Training burden
Moderate: read diffs, comment, approve, or request changes.
  1. Review starts from portal request, reminder, or Actions dispatch.
  2. A branch and pull request hold the change record and redline.
  3. Owner reviews the diff, comments inline, and confirms readiness.
  4. Merge publishes the record and updates audit evidence.
Role 03

Policy Governance Team

Full governance control through structured queues, registry updates, and approval gates.

Primary surface
GitHub Backend
GitHub required?
Yes
Training burden
Focused: routing, labels, registry, exceptions, evidence.
  • Maintain document registry, owners, stakeholder groups, and review dates.
  • Control approval labels that gate tier-appropriate merge authority.
  • Evaluate new document requests, exceptions, and conflict checks.
Handoff Pattern

The system reduces coordination by making handoffs explicit.

Employee finds or requests -> Owner or Submitter edits -> Policy Governance gates -> Published record updates
