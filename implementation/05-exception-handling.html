Workflow

  1. DMS-05.01 Requester opens exception issue with document reference, requirement, justification, mitigating controls, effective date, expiry date, and evidence.
  2. DMS-05.02 Action reads `document-registry.yaml` and assigns the Document Owner.
  3. DMS-05.03 Owner approval moves the issue to `exception-approved` and records the decision.
  4. DMS-05.04 Workflow commits an exception record to the document package and notifies Policy Team.
  5. DMS-05.05 Daily cron flags upcoming expiry and opens renewal issues.

Exit Criteria

  • DMS-05.06 Required fields cannot be bypassed.
  • DMS-05.07 Evidence attachment is retained or linked.
  • DMS-05.08 Expiry alerting produces test issues in a staging repo.
  • DMS-05.09 Exception records are visible in audit exports.
