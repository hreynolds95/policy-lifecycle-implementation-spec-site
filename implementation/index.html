Program Structure

Fourteen Tactical Workstreams

Each page expands one DocArchitect deliverable into implementation tasks, artifacts, dependencies, evidence, and exit criteria. The dashboard remains the executive status view; these pages are the build plan.

01 Architecture Overview

Component inventory, topology, and migration readiness scope.

02 Repository Taxonomy

Repo layout, domain taxonomy, metadata registry, and archive links.

03 Access Control

GitHub Teams, CODEOWNERS, SSO, and portal role gates.

04 Governance Workflow

Lifecycle states, QC, approvals, publication, and retirement.

05 Exception Handling

Issue workflow, owner approval, evidence, expiry, and renewal.

06 Template Enforcement

Six-section template checks and hard gate before Policy Team QC.

07 GitHub Pages Portal

Published library, search, role-gated requests, and conflict checks.

08 Awareness Communications

Stakeholder groups, material-change broadcasts, and monthly digest.

09 Orphaned Documents

Owner deactivation detection, alerts, issues, and onboarding follow-up.

10 Non-Governed Controls

Separate lightweight workflows for SOPs, job aids, and local procedures.

11 Gap Analysis

BRD requirement mapping, mitigations, and readiness gaps.

12 Legacy Migration

LogicGate transition, Google Drive archive links, and audit continuity.

13 Roadmap

Phased delivery plan, dependencies, sequencing, and acceptance milestones.

14 Open Decisions

Leadership, governance, security, tooling, and operating model choices.
