Two Surfaces, One System

The DMS separates the employee experience from the governance engine. Staff use a simple policy library portal; document owners, submitters, and the Policy Governance Team use GitHub to manage controlled changes, approvals, exceptions, and evidence.

Operating Model

The portal is the front door. GitHub is the control plane.

This split is what makes the DMS usable for the company while still satisfying governance, audit, and approval requirements. Most employees never need to learn GitHub. The controlled GitHub workflow remains available for the people accountable for document changes.

How Work Moves

A policy change can start simply and still end with a complete audit record.

01 Find or request

Employees use the portal to search the library, view current documents, or initiate a request.

02 Govern the change

GitHub creates the controlled record: issue, branch, pull request, review, approval, and merge.

03 Publish evidence

The merged source creates the published record, audit trail, PDF output, and notification evidence.
