Demo Objective

Show how the DMS replaces LogicGate with a governed GitHub workflow.

The demo should avoid deep technical detail up front. Start with the operating model, prove that governance controls remain intact, then route the group to the action plan for decisions, owners, and implementation sequencing.

Proof Points

What the audience should understand by the end

Control PoP tollgates remain visible.

GitHub does not remove governance. It makes routing, review, and approval evidence inspectable.

Experience Users do not need to live in GitHub.

The portal remains the simple entry point for staff, while governance teams use GitHub for controlled work.

Auditability Every published output traces to source.

The generated PDF, comments, approvals, checks, and publication record all connect back to the same commit.

Execution The plan is editable and assignable.

Implementation workstreams can be tracked through GitHub issues without sending broad notifications yet.

Decision Prompts

Questions to ask before ending the walkthrough

Approval model: Which PoP approval gates must be enforced by branch protection, CODEOWNERS, or issue-based approval?

Portal scope: Which user actions must be available without exposing staff to GitHub?

Cutover readiness: Which LogicGate records must be migrated before GitHub can become the authoritative record?

Operating model: Who owns metadata quality, exception renewal, orphan detection, and monthly portfolio review?
