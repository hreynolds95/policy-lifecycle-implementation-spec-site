09 Orphaned Document Management

Detect owner deactivation and alert Policy Team before review cycles are missed.

WorkplanPreviousNext

Detection Flow

  1. Monitor GitHub organization membership or IdP user lifecycle export.
  2. Match removed users against `document_owner` fields in the registry.
  3. Open Policy Team issue listing affected documents and next review dates.
  4. Post Slack alert and track replacement-owner onboarding.

Control Evidence

  • Test account removal event creates issue and Slack message.
  • Issue template captures replacement owner and onboarding date.
  • Escalation SLA is defined for documents near review deadlines.
