DMS Governance Gate Sequence

Every governed document change must clear a consistent set of automated gates before it becomes a published record. The sequence makes the PR the redline, the approval path, and the audit trail.

Gate Logic

The merge is blocked until the right evidence exists.

The DMS uses GitHub checks, ownership rules, tier routing, and approval labels to prevent uncontrolled publication. If a document has the wrong metadata, wrong submitter, wrong tier label, or missing approval, the workflow does not merge.

Approval Tiers

The label must match the document tier.

Tier 1 Board

Board-level approval evidence must be confirmed before the approval gate passes.

Tier 2 Committee

Committee path applies where the Policy on Policies requires intermediate governance approval.

Tier 3 Owner

Owner-level approval is accepted only where the document tier and change type allow it.
