DMS Live vs Phase 2 Readiness

This matrix translates the proof-of-concept brief into a stakeholder-ready view of what is already working, what remains for Phase 2, and which inputs are needed before a Block-wide implementation can move forward.

Readiness Lens

Separate proof-of-concept evidence from Phase 2 enterprise requirements.

The current PoC proves the operating model in a personal GitHub account. Phase 2 is the enterprise cutover path: move to the right org, enable SSO/SAML, confirm governance inputs, and migrate the full document inventory.

Live PoC 7

Portal, dashboard, Quincy, pilot docs, workflows, exceptions, and annual review dispatch.

Phase 2 4

Slack notifications, portal request initiation, SSO/login gate, and full migration.

Inputs Needed 3

IT, Legal/CCO, and Policy Governance decisions before enterprise implementation.

Feature Matrix

What is live today versus Phase 2

Capability Status What It Proves Phase 2 Need
Policy library portal with search, filters, document detail Live Staff can find and view compliance documents through a portal. Move from PoC hosting to enterprise access model.
Compliance metrics dashboard Live Review health, ownership, and document status can be summarized visually. Wire to authoritative registry and production reporting needs.
Quincy AI assistant Live Document corpus can support natural-language discovery and answers. Confirm approved enterprise AI integration pattern.
6 pilot documents with governance metadata Live Metadata model works for pilot policy records. Scale metadata quality checks to the full inventory.
12 governance workflows Live PoC GitHub can run the DMS control logic end-to-end. Migrate to enterprise GitHub org and production secrets.
Exception request form and lifecycle Live Exceptions can be requested, reviewed, retained, and renewed. Confirm migration approach for active LogicGate exceptions.
Annual review dispatch Live PoC Review process can start from workflow automation. Move from manual Actions trigger to fully automatic initiation.
Slack publication notifications Phase 2 Notification pattern is defined but webhook is not configured. Configure approved Slack webhook/channel and evidence retention.
Request Review portal button Phase 2 Desired owner experience is defined. Connect portal action to branch and PR creation workflow.
SSO / Block employee login gate Phase 2 Access boundary is known. Enable enterprise SSO/SAML and define employee access gates.
Full ~190 document migration Phase 2 Migration scope is understood. Clean inventory, map metadata, migrate content, and validate records.
Phase 2 Inputs

What must be confirmed before Phase 2 can begin

IT

Migrate repo to the enterprise GitHub org, enable GitHub SSO/SAML, and create compliance teams.

Legal / CCO

Confirm how Board-level approval evidence is applied and retained in GitHub.

Policy Governance

Confirm active exception migration and Markdown authoring guidance for document owners.
